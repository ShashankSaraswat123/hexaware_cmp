Infinity AppLens by LTIMindtree

Overview

Infinity AppLens is a cloud re-engineering assistant developed by LTIMindtree. It helps assess and prepare applications for cloud migration by analyzing the application code across various parameters. The tool provides insights into cloud compatibility and readiness, suggesting appropriate migration strategies.

Key Components and Features

Code Analysis

Analyzes application code across over 70 parameters, including session management, third-party dependencies, authentication methods, database connections, and hard-coded links.

Cloud Migration Assessment

Provides automated assessments for cloud enablement, identifying applications suited for refactoring, re-engineering, or redevelopment in cloud platforms like PaaS.

Customizable Parameters

Allows for custom configuration of analysis parameters, providing flexibility in assessing different applications.

Backend Tools and Technologies

Pluggable Architecture

Description: The tool features a pluggable architecture, enabling the addition of new features and assessment parameters as needed.

XML-Based Configuration

Description: Custom configuration of analysis parameters is done via XML, allowing for precise control over the assessment process.

Implementation Process

Initial Assessment

Infinity AppLens scans the application code from the repository and performs a comprehensive analysis based on predefined parameters.

Reporting

Generates detailed assessment reports, highlighting cloud readiness, potential migration complexities, and suggested strategies for cloud migration.

Decision Making

The insights provided assist in making informed decisions regarding cloud migration, including identifying applications for refactoring or redevelopment.

Simple Example of Infinity AppLens

Scenario: A retail company uses Infinity AppLens to evaluate its legacy e-commerce application for cloud migration. The tool identifies hard-coded links, outdated authentication methods, and third-party dependencies that need updating. Based on the assessment, the company decides to refactor the application to improve scalability and security, facilitating a smoother transition to a cloud-native architecture.
